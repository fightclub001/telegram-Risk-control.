# 🤖 Telegram 群组风控 & 反广告机器人

一个**开箱即用、可直接对外开放**的 Telegram 群管理机器人，用于自动检测和处理群组中的广告、垃圾信息与违规内容，适合**频道主 / 群主 / 机构运营**部署给他人使用。

## ✨ 功能总览

### 🔍 检测能力
- **简介检测**：检测用户个人简介中的链接和敏感词
- **显示名称检测**：检测显示名称中的敏感词
- **短消息检测**：检测连续发送极短消息（刷屏、灌水）行为
- **填充垃圾检测**：检测包含大量填充符号、空格的垃圾消息
- **语义广告相似度检测**（轻量版）：基于 `semantic_ads` 模块，对可疑广告内容做相似度匹配，识别「改文不改意」的广告

### 🎛️ 管理与控制台
- **统一配置面板**：所有关键参数可通过 `/admin` 管理员面板按钮化修改，无需改代码、无需重启
- **关键词管理**：支持简介敏感词的增删查，持久化存储
- **黑名单管理**：为各监控群配置独立的黑名单策略（启用/禁用、封禁时长）
- **统计与监控**：一键查看当前群组数量、管理员数量、举报记录数量、关键词数量、黑名单配置数量
- **一键数据备份**：在面板里点击即可将当前所有关键数据写入磁盘
- **多群组支持**：通过 `GROUP_IDS` 管理多个群组，配置共享，逻辑统一

### 🙋 用户交互体验
- 纯中文交互界面，面向中文用户友好
- 全程按钮式操作，无需记忆复杂命令
- 清晰的菜单结构，支持一键返回主菜单

## 📋 需求

- Python 3.9+
- aiogram >= 3.14.0
- Telegram Bot Token
- 管理员用户ID
- 监控群组ID

## 🚀 快速开始

### 本地运行

1. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

2. **设置环境变量**
   ```bash
   export BOT_TOKEN="your_bot_token_here"
   export ADMIN_IDS="123456789 987654321"
   export GROUP_IDS="111111111 222222222"
   ```

3. **运行机器人**
   ```bash
   python main.py
   ```

### Railway 部署

1. **创建 Railway 项目**
   - 在 [Railway.app](https://railway.app) 上创建新项目
   - 连接你的 GitHub 仓库

2. **配置环境变量**
   在 Railway 项目设置中添加：
   - `BOT_TOKEN`: 你的 Telegram Bot Token
   - `ADMIN_IDS`: 管理员用户ID（空格分隔）
   - `GROUP_IDS`: 群组ID（空格分隔）

3. **自动部署**
   - Push 到 GitHub 后，Railway 会自动部署
   - 机器人会在后台运行

## 📖 使用方法

### 管理员命令与面板结构

**入口命令：**

```text
/admin
```

在 Telegram 中**私聊**机器人并发送 `/admin`，即可打开管理员控制面板。

**主面板结构（对外用户能看到的界面）：**

```text
👑 管理员控制面板
├─ ⚙️ 配置管理
│  ├─ 🧹 清理任务
│  ├─ 📊 举报系统
│  ├─ ⚡ 速率限制
│  ├─ 🔍 关键词检测
│  ├─ 💬 消息管理
│  ├─ 🚫 黑名单
│  ├─ 📝 日志配置
│  └─ ⚙️ 性能配置
├─ 🔍 关键词管理
│  ├─ ➕ 添加关键词
│  ├─ ➖ 删除关键词（分页）
│  └─ 📋 查看所有关键词
├─ 🚫 黑名单管理
│  ├─ ⚙️ 配置黑名单策略
│  └─ 📋 查看各群黑名单配置（群名 + ID）
├─ 📊 统计信息
│  └─ 查看当前机器人整体运行数据
├─ 🔄 数据备份
│  └─ 立即写盘保存所有关键数据
└─ ← 返回机器人面板
```

只要你把机器人部署好并配置好 `ADMIN_IDS`，下游使用者只需要知道 `/admin` 这一个命令即可完成绝大部分配置与日常运维。

### 配置示例

#### 1. 编辑简介敏感词

1. `/admin` → `⚙️ 群组管理` → 选择群组
2. `🔍 简介检测设置` → `📋 编辑敏感词列表`
3. 输入敏感词（一行一个），例如：
   ```
   qq:
   微信
   加我
   福利
   ```

#### 2. 调整短消息检测参数

1. `/admin` → 选择群组 → `💬 短消息检测`
2. 调整各项参数：
   - 字数阈值：短于此长度的消息被视为极短消息（默认3）
   - 连续条数：连续发送极短消息的数量（默认2）
   - 时间窗口：检测时间范围（默认60秒）

#### 3. 启用自动回复

1. `/admin` → 选择群组 → `🤖 自动回复设置`
2. 启用自动回复
3. 添加关键词（如："hello", "hi"）
4. 设置回复文本
5. 可选：添加按钮和删除延时

## 🔧 配置参数说明

### 简介检测
- `check_bio_link`: 是否检测简介中的链接（http://, https://, t.me/, @）
- `check_bio_keywords`: 是否检测简介中的敏感词
- `bio_keywords`: 敏感词列表

### 显示名称检测
- `check_display_keywords`: 是否检测显示名称中的敏感词
- `display_keywords`: 敏感词列表

### 短消息检测
- `short_msg_detection`: 启用/禁用
- `short_msg_threshold`: 极短消息的字符数阈值（默认3）
- `min_consecutive_count`: 需要连续发送的次数（默认2）
- `time_window_seconds`: 时间窗口（默认60秒）

### 填充垃圾检测
- `fill_garbage_detection`: 启用/禁用
- `fill_garbage_min_raw_len`: 检测的最小原始消息长度（默认12）
- `fill_garbage_max_clean_len`: 清理后的最大长度（默认8）
- `fill_space_ratio`: 空格/特殊字符的比例阈值（默认0.30）

### 自动回复
- `enabled`: 启用/禁用自动回复
- `keywords`: 触发关键词列表
- `reply_text`: 回复文本
- `buttons`: 按钮列表
- `delete_user_sec`: 用户消息删除延时（0=不删）
- `delete_bot_sec`: 机器人消息删除延时（0=不删）

### 重复发言与举报
- **重复发言**：时间窗口、触发次数、首次禁言时长；当处罚阈值为 4 条时，第二次重复命中仅记录、第三次提醒、第四次处罚；**豁免词(白名单)**：消息包含任一词则不触发重复检测，一行一个可配置
- **备用群组发图校验**：每个主群都可以单独开启/关闭，默认关闭。开启后，管理员可在「媒体权限」里勾选必备备用群组；未加入这些群的成员发送媒体会被删除，并收到“发图请先加入备用群组！”和「一键加入」按钮
- **举报按钮规则**：管理员未点击封禁/误判豁免前按钮永不过期（不因原消息被删而移除）；机器人自动封禁或管理员封禁时移除按钮并保留/更新文案；误判豁免时删除警告消息；超过 24 小时未处理则仅隐藏按钮、保留文案

## 📁 数据文件

数据目录优先读取环境变量 `DATA_DIR`（兼容旧变量 `CONFIG_DIR`）。

- Docker 默认：`/app/data`
- Ubuntu 本地 systemd 推荐：`/opt/telegram-risk-control/data`
- 未设置时会回落到默认数据目录

数据文件示例：
- `config.json`: 所有群组配置
- `reports.json`: 举报记录
- `backups/config/config-*.json`: 配置自动备份快照（默认保留 120 份）

### 一键加入页入口
- `MEDIA_JOIN_GATE_MINIAPP_URL`：备用群组加入页地址。机器人会把当前主群的必备群组数据编码进 URL 查询参数，页面据此渲染「一键加入」入口
- 默认公开地址：`https://telegarmpromax.com/join-gate.html`
- 如果未显式设置环境变量，机器人会优先回落到这个 Cloudflare Pages 地址
- 群内提醒按钮使用普通 URL 按钮打开该页面，避免 Telegram Web App 按钮在群聊场景不可用

## ⚙️ 环境变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `BOT_TOKEN` | Telegram Bot Token | `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11` |
| `ADMIN_IDS` | 管理员用户ID（空格分隔） | `123456789 987654321` |

**获取群组ID的方法：**
1. 在群组中发送任何消息
2. 使用机器人转发消息
3. 检查转发消息的ID

## 🐛 故障排查

### 机器人无法连接
- 检查 `BOT_TOKEN` 是否正确
- 检查网络连接

### 机器人无法发送消息
- 确保机器人是群组管理员
- 检查机器人是否被禁言
- 确保配置的群组ID正确

### 管理命令不工作
- 检查用户ID是否在 `ADMIN_IDS` 中
- 确保发送 `/admin` 命令时是在私聊中

### 检测失效
- 检查群组是否在 `GROUP_IDS` 中
- 确保群组检测功能未被禁用
- 查看日志输出
