# 🤖 Telegram 群组管理机器人

一个功能完整的 Telegram 机器人，用于检测和管理群组中的广告和垃圾信息。

## ✨ 功能特性

### 🔍 检测功能
- **简介检测**：检测用户个人简介中的链接和敏感词
- **显示名称检测**：检测显示名称中的敏感词
- **短消息检测**：检测连续发送极短消息的行为
- **填充垃圾检测**：检测包含大量填充符号和空格的消息

### 🎛️ 管理功能
- **完全可配置**：所有检测参数都可通过管理员面板调整
- **群组管理**：支持多个群组，每个群组独立配置
- **自动回复**：支持设置关键词自动回复
- **用户豁免**：支持为合法用户豁免检测
- **举报系统**：用户可举报违规用户，管理员可执行封禁

### 🌐 交互方式
- 纯中文交互界面
- 按钮式操作，无需记忆命令
- 清晰的菜单结构，支持返回按钮

## 📋 需求

- Python 3.9+
- aiogram >= 3.14.0
- Telegram Bot Token
- 管理员用户ID
- 监控群组ID

## 🚀 快速开始

### 本地运行

1. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

2. **设置环境变量**
   ```bash
   export BOT_TOKEN="your_bot_token_here"
   export ADMIN_IDS="123456789 987654321"
   export GROUP_IDS="111111111 222222222"
   ```

3. **运行机器人**
   ```bash
   python main.py
   ```

### Railway 部署

1. **创建 Railway 项目**
   - 在 [Railway.app](https://railway.app) 上创建新项目
   - 连接你的 GitHub 仓库

2. **配置环境变量**
   在 Railway 项目设置中添加：
   - `BOT_TOKEN`: 你的 Telegram Bot Token
   - `ADMIN_IDS`: 管理员用户ID（空格分隔）
   - `GROUP_IDS`: 群组ID（空格分隔）

3. **自动部署**
   - Push 到 GitHub 后，Railway 会自动部署
   - 机器人会在后台运行

### 配置示例

#### 1. 编辑简介敏感词

1. `/admin` → `⚙️ 群组管理` → 选择群组
2. `🔍 简介检测设置` → `📋 编辑敏感词列表`
3. 输入敏感词（一行一个），例如：
   ```
   qq:
   微信
   加我
   福利
   ```

#### 2. 调整短消息检测参数

1. `/admin` → 选择群组 → `💬 短消息检测`
2. 调整各项参数：
   - 字数阈值：短于此长度的消息被视为极短消息（默认3）
   - 连续条数：连续发送极短消息的数量（默认2）
   - 时间窗口：检测时间范围（默认60秒）

#### 3. 启用自动回复

1. `/admin` → 选择群组 → `🤖 自动回复设置`
2. 启用自动回复
3. 添加关键词（如："hello", "hi"）
4. 设置回复文本
5. 可选：添加按钮和删除延时

## 🔧 配置参数说明

### 简介检测
- `check_bio_link`: 是否检测简介中的链接（http://, https://, t.me/, @）
- `check_bio_keywords`: 是否检测简介中的敏感词
- `bio_keywords`: 敏感词列表

### 显示名称检测
- `check_display_keywords`: 是否检测显示名称中的敏感词
- `display_keywords`: 敏感词列表

### 短消息检测
- `short_msg_detection`: 启用/禁用
- `short_msg_threshold`: 极短消息的字符数阈值（默认3）
- `min_consecutive_count`: 需要连续发送的次数（默认2）
- `time_window_seconds`: 时间窗口（默认60秒）

### 填充垃圾检测
- `fill_garbage_detection`: 启用/禁用
- `fill_garbage_min_raw_len`: 检测的最小原始消息长度（默认12）
- `fill_garbage_max_clean_len`: 清理后的最大长度（默认8）
- `fill_space_ratio`: 空格/特殊字符的比例阈值（默认0.30）

### 自动回复
- `enabled`: 启用/禁用自动回复
- `keywords`: 触发关键词列表
- `reply_text`: 回复文本
- `buttons`: 按钮列表
- `delete_user_sec`: 用户消息删除延时（0=不删）
- `delete_bot_sec`: 机器人消息删除延时（0=不删）

## 📁 数据文件

数据会自动保存在 `/data` 目录：

机器人需要以下权限：
- ✅ 发送消息
- ✅ 编辑消息
- ✅ 删除消息
- ✅ 限制成员（禁言）

## 🐛 故障排查

### 机器人无法连接
- 检查 `BOT_TOKEN` 是否正确
- 检查网络连接

### 机器人无法发送消息
- 确保机器人是群组管理员
- 检查机器人是否被禁言
- 确保配置的群组ID正确

### 管理命令不工作
- 检查用户ID是否在 `ADMIN_IDS` 中
- 确保发送 `/admin` 命令时是在私聊中

### 检测失效
- 检查群组是否在 `GROUP_IDS` 中
- 确保群组检测功能未被禁用
- 查看日志输出
