# ✅ 部署前检查清单

使用此清单确保一次部署成功，避免问题。

## 📋 准备阶段

### 信息收集

- [ ] **BOT_TOKEN**
  - [ ] 从 [@BotFather](https://t.me/botfather) 获取
  - [ ] 格式检查：`123456:ABCDEFghijklmnopqrstuvwxyz...`
  - [ ] 复制到安全地方（不要分享）
  - [ ] ✅ 确认有效性

- [ ] **ADMIN_IDS**
  - [ ] 从 [@userinfobot](https://t.me/userinfobot) 获取
  - [ ] 获取至少一个管理员 ID
  - [ ] 格式：纯数字（如 `123456789`）
  - [ ] ✅ 至少有一个有效 ID

- [ ] **GROUP_IDS**
  - [ ] 识别需要监控的群组
  - [ ] 从 [@userinfobot](https://t.me/userinfobot) 获取群组 ID
  - [ ] 格式：负数或正数（如 `-1001234567890`）
  - [ ] ✅ 确认至少一个有效群组 ID

### 账户准备

- [ ] **GitHub 账户**
  - [ ] 已登录 GitHub
  - [ ] 有创建仓库的权限
  - [ ] ✅ 准备好仓库名称

- [ ] **Railway 账户**
  - [ ] 已注册 Railway
  - [ ] 已验证邮箱
  - [ ] 已授权 GitHub 访问
  - [ ] ✅ 账户可用

## 🔗 Railway 部署阶段

### 创建项目

- [ ] **新建项目**
  - [ ] 进入 Railway.app
  - [ ] 点击 "New Project"
  - [ ] 选择 "Deploy from GitHub repo"
  - [ ] 授权 GitHub 访问
  - [ ] ✅ 授权完成

- [ ] **选择仓库**
  - [ ] 在列表中找到 `telegram-bot`
  - [ ] 点击仓库名称
  - [ ] ✅ 选择完成

- [ ] **配置部署**
  - [ ] 点击 "Deploy Now"
  - [ ] 等待部署开始
  - [ ] ✅ 部署已启动

### 环境变量配置

- [ ] **添加 BOT_TOKEN**
  - [ ] 点击 "Variables" 标签
  - [ ] 点击 "Add Variable"
  - [ ] 名称：`BOT_TOKEN`
  - [ ] 值：你的 Token
  - [ ] 点击 "Add"
  - [ ] ✅ 已添加

- [ ] **添加 ADMIN_IDS**
  - [ ] 点击 "Add Variable"
  - [ ] 名称：`ADMIN_IDS`
  - [ ] 值：你的管理员 ID（空格分隔）
  - [ ] 点击 "Add"
  - [ ] ✅ 已添加

- [ ] **添加 GROUP_IDS**
  - [ ] 点击 "Add Variable"
  - [ ] 名称：`GROUP_IDS`
  - [ ] 值：你的群组 ID（空格分隔）
  - [ ] 点击 "Add"
  - [ ] ✅ 已添加

- [ ] **验证环境变量**
  - [ ] 检查三个变量都已显示
  - [ ] 值不为空
  - [ ] 格式正确
  - [ ] ✅ 环保变量验证完成

### 部署监控

- [ ] **查看部署状态**
  - [ ] 点击 "Logs" 标签
  - [ ] 查看实时日志
  - [ ] ✅ 日志正在输出

- [ ] **等待部署完成**

## 🔧 故障排查检查

如果遇到问题，按此检查：

### 机器人无响应

- [ ] **BOT_TOKEN 检查**
  - [ ] Token 是否正确
  - [ ] Token 是否包含冒号
  - [ ] Token 是否过期
  - [ ] ✅ Token 有效

- [ ] **日志检查**
  - [ ] 查看 Railway 日志
  - [ ] 查找错误信息
  - [ ] ✅ 识别问题

### /admin 不工作

- [ ] **权限检查**
  - [ ] 你的 ID 在 `ADMIN_IDS` 中吗？
  - [ ] 在私聊中发送的吗？
  - [ ] 等待 30 秒后重试？
  - [ ] ✅ 排查完成

### 群组检测无效

- [ ] **配置检查**
  - [ ] 群组 ID 在 `GROUP_IDS` 中吗？
  - [ ] 机器人是管理员吗？
  - [ ] 群组功能是否启用？
  - [ ] ✅ 配置正确
